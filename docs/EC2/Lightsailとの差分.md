# Lightsail との差分（開発者向け）

> EC2移行に伴い「Lightsail時代と何が変わり、何が変わらないか」を開発者・運用者向けに整理。
> 管理番号 202607241629 / 作成 2026-07-24。

---

## 0. 要点（先に結論）

- **ローカル開発フローは一切変わらない**（`docker compose up -d --build` も、テストも、改修の進め方も同じ）。
- 変わるのは**本番反映の「宛先」と「接続方法」だけ**（Lightsailコンソール → PCからの鍵SSH、旧IP → 新EIP）。
- **改修依頼の書き方（`/改修` テンプレート・管理番号の自動採番・完了セルフチェック）は変更なし。**
- ⚠ 現在は**二重稼働**（Lightsail=現行 / EC2=移行先で検証中）。切替は未実施。

## 1. 変更点の一覧

| 項目 | 旧: Lightsail | 新: EC2 |
|---|---|---|
| ホスティング | AWS Lightsail（2GB/2vCPU/60GB・$12/月固定） | AWS EC2 `i-0ce3a2e284f376401`（t3.small 2GB+swap4GB / gp3 30GB） |
| IP | 静的IP **52.197.43.164** | Elastic IP **52.199.22.60** |
| OS | Ubuntu 22.04 LTS | **Ubuntu 24.04 LTS**（コンテナ構成のため差異は無影響） |
| SSH | Lightsailコンソールのブラウザ SSH（または ubuntu 鍵） | **PCから鍵SSH**：`ssh -i tutor-ec2-key.pem ubuntu@52.199.22.60`（キーペア `tutor-ec2-key`） |
| アクセスURL（現在） | http://52.197.43.164:8000 / :8001 | http://52.199.22.60:8000 / :8001 |
| バックアップ | 自動化なし（手動 `pg_dump` のみ） | S3バケット `tutor-report-system-backup-nxtech2026` ＋ IAMロール（**準備済／cron未実装**） |
| IAM | なし | ロール `tutor-ec2-s3-backup-role` を EC2 にアタッチ（キー不要でS3書込） |
| 配置ディレクトリ | `~/tutor-report-system` | **同じ** `~/tutor-report-system` |
| デプロイ | `git pull && sudo docker compose up -d --build` | **同じコマンド**（SSH接続方法のみ変更） |
| メール | `MAIL_BACKEND=console`（送信オフ） | **同じ** `console`（送信オフ・実メール飛ばない） |
| DB | コンテナ内 `postgres:16-alpine`（共有 `tutor`） | **同じ**（RDS未移行・`DATABASE_URL` 一本） |
| ドメイン/HTTPS | 未取得・未対応 | 未切替・未対応（`kintai-*.haken.net` は将来） |

## 2. 変わらないもの（開発で意識しなくてよい）

- **ローカル開発**: `cp .env.example .env` → `docker compose up -d --build` → `pytest`。手順・ポート（8000/8001/8025）とも従来どおり。
- **コード・DBスキーマ・テスト**: 変更なし（移行はインフラのみ・アプリコードは無改変）。
- **改修フロー**: `/改修`（対象システム＋依頼内容）→ 管理番号自動採番 → 実装 → 両システムフル pytest（`MAIL_BACKEND=console`）→ commit/push → 本番反映。**すべて不変**。
- **共有リソースの規約**: `.env` 共有、`users`/`assignments`/`invitations` 共有、メールキュー直列化、複製ペア同時更新 — すべて従来どおり。

## 3. 開発者が新たに知っておくこと

1. **本番SSHが鍵方式になった**: `tutor-ec2-key.pem` が無いとEC2に入れない（Lightsailのブラウザ SSH は使えない）。鍵は作業者PCローカル保管・リポジトリやチャットに置かない。
2. **旧IP `52.197.43.164` は“新本番”ではない**: ドキュメント・ブックマーク・監視で旧IPを参照している箇所は新EIP `52.199.22.60` へ更新（ただし切替までは旧も現行として生きている＝下記4）。
3. **本番反映先の正**: 接続情報は `docs/INFRASTRUCTURE.md`、操作は `docs/EC2/デプロイ手順書.md`。

## 4. ⚠ 二重稼働に関する注意（切替まで）

- **Lightsail（52.197.43.164）と EC2（52.199.22.60）が同時に稼働**しており、**それぞれ独立したDBを持つ**。
- EC2 側で行った操作・投入データは **Lightsail に反映されない**（逆も同様）。EC2のDBは移行時点のコピー。
- **どちらを触っているか常に意識**する。検証はEC2、現行ユーザー運用はLightsail。
- **本番切替時に最終データ同期が必須**（Lightsailの最新を `pg_dump`→EC2へ復元）。手順は `デプロイ手順書.md` §8。
- 切替が済むまで、EC2で `seed_production --yes` 等の破壊的操作をしても現行ユーザーには影響しないが、**逆に「EC2で検証したデータ」は切替時に上書きされうる**点に注意。

## 5. セキュリティ上の差分（現状の注意）

| 項目 | 状態 |
|---|---|
| DB(5432)・MailHog(8025/1025) の外部公開 | ⛔ 閉（EC2で適切に非公開・良好） |
| アプリ(8000/8001) | ⚠ 平文HTTPで外部開放（HTTPS化後にSGで閉じる予定） |
| 通信暗号化 | ⚠ 未（HTTPS未対応）。ログイン情報・JWTが平文で流れるため、HTTPS化まで公開範囲に留意 |
