## 1. システム概要

**イスト勤怠レポート for 代々木進学会（旧称: 指導実績報告システム）** は、講師が指導実績を登録し、保護者が月ごとに確認し、運営が受付・再鑑（＝最終承認）まで進めるためのシステムです。URL は `http://localhost:8000` です。

| 登場人物 | 役割 | 主な操作 |
|---|---|---|
| 講師 | 指導後に報告書を作成し、月ごとに保護者へ承認依頼します。 | 報告書作成、修正、削除、保護者への承認依頼 |
| 保護者 | 講師から依頼された報告書を月ごとに確認します。承認すると運営へ自動提出されます。 | 月次確認、まとめて承認、差戻し |
| 運営 | 提出された報告書を講師、生徒、月の単位で確認します。 | 受付、再鑑（＝最終承認）、差戻し、ユーザー管理、未処理クローズ |

運営には次の役割があります。承認フローに加わるのは **運営受付** と **運営再鑑** で、**運営再鑑の承認がそのまま最終承認**になります。**運営管理者・運営管理責任者は承認フローの外**にあり、最終承認は行いません（閲覧・PDF・ユーザー/担当管理・未処理クローズを担当します）。

| 運営ロール | 役割 |
|---|---|
| 運営受付（admin_receiver） | 講師から運営へ提出された報告書を受け付けます。 |
| 運営再鑑（admin_reviewer） | 受付済みの報告書を再確認します。**この再鑑承認が最終承認**です。完了後に修正が必要なときは受付へ差し戻せます。 |
| 運営管理者（admin_master） | 承認フロー外。報告書の閲覧・PDF、ユーザー作成、講師と保護者の紐付け、未処理報告のクローズを行います。 |
| 運営管理責任者（admin_chief） | 承認フロー外。運営管理者の操作に加え、管理責任者専用の設定（保護者承認スキップ設定・管理責任者の招待/運用）を行います。 |

## 2. ログインログアウト

### ログイン手順

1. ブラウザで `http://localhost:8000/login` を開きます。
2. メールアドレスを入力します。
3. パスワードを入力します。
4. `ログイン` ボタンを押します。
5. ログインに成功すると、自分の役割に合った画面へ移動します。

### ログアウト手順

1. 画面上部の `ログアウト` ボタンを押します。
2. ログイン画面に戻ればログアウト完了です。

### パスワードを忘れた場合

1. ログイン画面を開きます。
2. `パスワードをお忘れの方はこちら` を押します。
3. 登録済みのメールアドレスを入力します。
4. `リセットメールを送る` を押します。
5. `メールを送信しました。ご確認ください。` と表示されたら、メールを確認します。
6. メール本文のパスワードリセットURLを開きます。
7. 新しいパスワードと確認用パスワードを入力します。
8. `パスワードを変更する` を押します。
9. `パスワードを変更しました` と表示され、3秒後にログイン画面へ移動します。
10. 新しいパスワードでログインします。

リセットURLの有効期限は1時間です。期限切れまたは使用済みの場合は、もう一度リセットメールを送信してください。開発環境ではメールは MailHog（`http://localhost:8025`）で確認します。

## 4. ロール別 操作手順

### 講師（tutor）

#### 報告書の新規作成手順

1. 講師アカウントでログインします。
2. `http://localhost:8000/tutor/reports` を開きます。
3. 右側の `簡易作成` フォームに入力します。
4. 生徒を選びます。
5. 指導日、開始時刻、終了時刻、休憩等の時間、科目、指導内容を入力します。
6. 指導時間数が自動で計算されることを確認します。
7. `保存` ボタンを押します。
8. 一覧に報告書が追加されれば作成完了です。

#### 新しい生徒を追加して報告書を作成する手順

1. `http://localhost:8000/tutor/reports` を開きます。
2. 生徒プルダウンで `＋ 新しい生徒を追加` を選びます。
3. 表示された生徒名入力欄に生徒名を入力します。
4. 指導日、時刻、科目、指導内容を入力します。
5. `保存` ボタンを押します。
6. 生徒が登録され、その生徒の報告書が作成されます。

新しく追加した生徒は、最初は保護者未連携です。保護者と紐付ける場合は、運営管理者が後から紐付けを行います。

#### 報告書の確認、編集、削除

1. `http://localhost:8000/tutor/reports` を開きます。
2. 月選択プルダウンで確認したい月を選びます。
3. 生徒プルダウンで確認したい生徒を選びます。
4. 下書きの報告書は `編集` と `削除` ができます。
5. 差戻し中の報告書は `編集` ができます。
6. 過去月や提出済みの報告書は、基本的に参照のみです。

#### 保護者へ月次承認依頼する手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. 対象月のカードを確認します。
3. その月の報告書がすべて下書きから進められる状態であれば、`保護者へまとめて承認依頼を送る` ボタンが表示されます。
4. ボタンを押します。
5. 保護者側の状態が `承認待ち` になれば依頼完了です。

#### 差戻されたときの修正再依頼手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. 対象月に `差戻しがあります` と表示されていることを確認します。
3. `http://localhost:8000/tutor/reports` を開きます。
4. 差戻し理由を確認します。
5. 対象の報告書を編集します。
6. 修正後、再度 `http://localhost:8000/tutor/approval` から保護者へ承認依頼します。

#### 講師のエクスポート手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. エクスポートしたい月を選びます。
3. 生徒別に出力する場合は、最終承認済みカードの `指導時間確認票PDF` または `指導日報PDF` を押します。
4. 対象月の全生徒をまとめて出力する場合は、上部の `指導時間確認票PDF` / `指導日報PDF` ボタンを押します。

講師の一括エクスポートは、その月の対象生徒がすべて最終承認済みの場合のみ有効です。

PDFは2種類あります。

| ボタン | 内容 | ファイル名 |
|--------|------|-----------|
| 指導時間確認票PDF | 勤務表（回数・指導日・在室した時間帯・指導時間数の月間一覧と月計） | `指導実績_生徒名_YYYY年MM月.pdf` |
| 指導日報PDF | 紙の指導日報様式（1ページ5日分）。日ごとの教科・使用教材・指導内容・学習状況・宿題状況・次回の予定を記載し、保護者承認済みの指導日には会員認め印（電子印）が印字されます | `指導日報_YYYY年MM月.pdf` |

#### 運営提出について

現在の実装では、保護者が月次承認するとシステムが自動で運営へ提出します。講師が保護者承認後に別途 `運営へ提出` を押す運用ではありません。

`/tutor/reports` には個別・一括の運営提出 API を呼ぶ古い操作導線が一部残っていますが、通常運用では `/tutor/approval` で保護者依頼までを行い、その後は保護者承認を待ちます。

### 保護者（parent）

#### 承認依頼を確認する手順

1. 保護者アカウントでログインします。
2. `http://localhost:8000/parent/approval` を開きます。
3. 画面上部の月選択プルダウンで確認したい月を選びます。
4. 選択した月の報告書だけが表示されます。
5. 確認したいカードの `報告書を確認` を押すと、報告書（参照）画面で指導日、在室時間、指導時間数、科目、指導内容を確認できます。

未来月は月選択に表示されません。過去月は参照のみで、承認や差戻しはできません。

#### 月次承認する手順

1. `http://localhost:8000/parent/approval` を開きます。
2. 月選択で当月を選びます。
3. 対象カードの `報告書を確認` を押し、報告書（参照）画面で内容を確認します。
4. 問題がなければ、参照画面の `承認する`（`5月分を承認する` のような表記）ボタンを押します。
5. 確認ダイアログで内容を確認し、承認します。
6. 表示が `承認済み` または運営側の `受付待ち` に進めば完了です。承認と同時に運営へ自動提出されます。

#### 月次差戻し手順

1. `http://localhost:8000/parent/approval` を開きます。
2. 月選択で当月を選びます。
3. 対象カードの `報告書を確認` を押し、報告書（参照）画面を開きます。
4. 内容に修正が必要な場合は `差戻す`（`5月分を差戻す` のような表記）ボタンを押します。
5. 差戻し理由を入力します。
6. 差戻しを実行します。
7. 対象月の承認待ち報告書がまとめて講師へ差戻され、講師側に差戻し理由が表示されます。

差戻し理由は、講師が修正内容を理解できるように具体的に書いてください。

#### 保護者のエクスポート手順

1. `http://localhost:8000/parent/approval` を開きます。
2. 月と生徒を選びます。
3. 最終承認済みの月では、`指導時間確認票PDF` と `指導日報PDF` のダウンロードボタンが表示されます。
4. 生徒を選択せずに全生徒をまとめて出力することもできます。

指導日報PDFには、日ごとの指導内容とあわせて、保護者承認済みの指導日に会員認め印（電子印）が印字されます。

### 運営受付（admin_receiver）

#### 受付待ち一覧の確認手順

1. 運営受付アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/receive` を開きます。
3. 月選択プルダウンで対象月を選びます。
4. 講師選択プルダウンで対象講師を選びます。全員を確認する場合は `すべての講師` を選びます。
5. 生徒ごとのカードを確認します。
6. カードのステータスが `受付待ち` のものが受付対象です。

#### 受付処理の手順

1. 受付待ちの生徒カードを確認します。
2. `報告書を確認` を押し、報告書（参照）画面で明細を確認します。
3. 問題がなければ参照画面の `受付する`（`5月分を受付する` のような表記）ボタンを押します。
4. 確認ダイアログで内容を確認し、受付します。
5. カードのステータスが `再鑑待ち` になれば受付完了です。

#### 月次差戻し手順

1. 受付待ちの生徒カードで `報告書を確認` を開きます。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. 対象生徒・対象月の報告書がまとめて `差戻し中` になれば差戻し完了です。

#### 運営のエクスポート手順

1. `http://localhost:8000/admin/dashboard` を開きます。
2. 月選択と講師選択で対象を絞り込みます。
3. 最終承認済みの生徒カードを開くと、`指導時間確認票PDF` と `指導日報PDF` を生徒別に出力できます。
4. 上部の `完了分PDF（指導時間確認票）` / `完了分PDF（指導日報）` ボタンを押すと、対象月の最終承認済み分を一括出力できます。

### 運営再鑑（admin_reviewer）

運営再鑑の承認が、このシステムの **最終承認** です。

#### 再鑑待ち一覧の確認手順

1. 運営再鑑アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/review` を開きます。
3. 月選択と講師選択を使って対象を絞り込みます。
4. 生徒ごとのカードを確認します。
5. カードのステータスが `再鑑待ち` のものが再鑑対象です。

#### 再鑑（最終承認）処理の手順

1. 再鑑待ちの生徒カードを確認します。
2. `報告書を確認` を押し、報告書（参照）画面で明細を確認します。
3. 問題がなければ参照画面の `再鑑する`（`5月分を再鑑する` のような表記）ボタンを押します。
4. 確認ダイアログで内容を確認し、再鑑します。
5. カードのステータスが `最終承認済み` になれば最終承認完了です。講師・保護者へ最終承認の通知メールが送られます。

#### 月次差戻し手順

1. 再鑑待ちの生徒カードで `報告書を確認` を開きます。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. 対象生徒・対象月の報告書がまとめて受付へ差戻され（`受付へ差戻し中`）、受付担当が再受付します。

#### 最終承認後にやり直したい場合

最終承認（完了）後でも、内容に誤りが見つかった場合は運営再鑑が差し戻せます。再鑑待ち／完了のカードから `差戻し` を行うと、報告書は受付へ戻ります（`受付へ差戻し中`）。受付担当が再受付し、再度再鑑（最終承認）まで進めます。

### 運営管理者（admin_master）／運営管理責任者（admin_chief）

運営管理者・運営管理責任者は **承認フローの外** にあります。最終承認は行いません（最終承認は運営再鑑が行います）。これらのロールは、報告書の閲覧・PDF、ユーザー作成・管理、講師と保護者の紐付け、未処理報告のクローズを担当します。運営管理責任者は、これらに加えて管理責任者専用の設定（保護者承認スキップ設定・管理責任者の招待/運用）を行えます。

> 注意: 運営管理者・運営管理責任者には承認キュー（受付待ち／再鑑待ち／最終承認待ち）の操作画面はありません。`/admin/dashboard` と報告書（参照）画面は閲覧専用として利用できます（承認・差戻しボタンは表示されません）。

#### 報告書を閲覧する手順

1. 運営管理者（または運営管理責任者）アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` を開きます。
3. 月選択と講師選択で対象を絞り込みます。
4. 生徒カードの `報告書を確認` を押すと、報告書（参照）画面で内容を確認できます（参照のみ）。
5. エクスポート（`指導時間確認票PDF` / `指導日報PDF`）も同画面から行えます。

#### 未処理報告のクローズ手順

1. `http://localhost:8000/admin/stale-reports` を開きます。
2. 先月以前で未完了の報告書（未処理報告）の一覧を確認します。
3. クローズ対象の報告書で、クローズ理由を入力してクローズします（理由は必須）。

> 未処理報告のクローズは、運営受付・運営再鑑・運営管理者・運営管理責任者のいずれも実行できます。

#### ユーザー追加手順（招待方式）

1. 運営管理者アカウントでログインします。
2. `http://localhost:8000/admin/users` を開きます。
3. 新規ユーザー登録欄でロールを選びます。
4. 講師・運営スタッフの場合は氏名とメールアドレスを入力します。
5. 保護者の場合はメールアドレス、担当講師、生徒名を入力します。氏名は登録時に自動設定されます。
6. `招待メールを送る` を押します。
7. 登録済みユーザー一覧に「未登録」として表示され（招待済み一覧は廃止し、登録済みユーザー一覧に統合）、招待先ユーザーがメールのリンクからパスワードを設定すると登録完了です。

講師番号は講師招待時に自動採番されます。招待リンクの有効期限は72時間です。未登録の招待は一覧の `再送` または `取消` ができます。登録済みユーザーは詳細から有効化・無効化できます。

> ユーザー管理画面（`/admin/users`）は、運営受付・運営再鑑も運営管理者と同等に利用できます。

#### 保護者承認スキップを設定する手順

旧「システム設定」画面は廃止しました。保護者承認のスキップはユーザー管理画面で設定します。

1. 運営管理者（または運営管理責任者）アカウントでログインします。
2. `http://localhost:8000/admin/users` を開きます。
3. 対象の保護者ユーザーの `詳細` を開きます。
4. 「保護者承認設定」の `保護者承認をスキップする` をオンにします。
5. オンにすると、その保護者の担当報告書は保護者承認を省略して直接運営へ提出されるようになります。

担当紐付け（既存生徒への講師追加）は、講師の生徒管理画面と保護者招待（`/admin/users`）から作成します。リマインドは講師が承認管理画面（承認依頼時）で設定します。

## 5. リマインド通知について

リマインド通知は、月末が近づいたときに自動で作成されます。標準設定では、月末の3日前から月末当日までが対象です。

| 状態 | 通知先 | 内容 |
|---|---|---|
| 保護者承認待ち | 保護者 | 未承認の報告書があることを知らせます。 |
| 下書きのまま | 講師 | まだ提出されていない報告書があることを知らせます。 |
| 差戻し後、未再提出 | 講師 | 修正と再提出が必要な報告書があることを知らせます。 |
| 保護者承認済みだが運営未提出 | 講師 | 通常は保護者承認時に自動提出されるため短時間のみ発生します。残っている場合は運営への提出が必要な報告書があることを知らせます。 |

開発環境では、送信されたメールは `http://localhost:8025` の MailHog で確認できます。

## 6. よくある操作ミスと対処法

### 誤って提出してしまった場合

提出後は、講師側で自由に削除や修正ができない状態になります。内容に誤りがある場合は、保護者または運営に連絡し、差戻しを依頼してください。

差戻し後、講師は内容を修正して再提出できます。

### 差戻しコメントを入力し忘れた場合

差戻しには理由コメントが必要です。コメントを入力しないと、差戻しは完了しません。

コメント未入力でエラーになった場合は、もう一度差戻し操作を行い、理由を入力してから送信してください。

例として、次のように具体的に書くと相手が対応しやすくなります。

「指導時間が実際と異なるため修正してください」

「科目名が未入力のため追記してください」

### 月を間違えて見ている場合

報告書が見つからない場合は、画面上部の月選択を確認してください。保護者画面と運営画面は、選択した1か月分だけを表示します。

### 過去月でボタンが表示されない場合

過去月は参照のみです。承認、差戻し、受付、再鑑、最終承認のボタンは表示されません。

### 最終承認のボタンが見当たらない場合

最終承認は **運営再鑑（admin_reviewer）** が再鑑待ちのカードで行います。運営管理者・運営管理責任者は承認フローの外のため、最終承認のボタンは表示されません。最終承認が必要な場合は、運営再鑑アカウントで `/admin/queue/review` を開いてください。

## 7. 管理者向け：起動停止手順

以下のコマンドは、`C:\PerformanceReportingSystem\tutor-report-system` フォルダで実行してください。

### 起動

```powershell
docker compose up -d --build
```

### 停止

```powershell
docker compose down
```

### 再起動

```powershell
docker compose restart backend
```

### マイグレーション適用

```powershell
docker compose exec backend alembic upgrade head
```

### シードデータ投入

```powershell
docker compose exec backend python -m app.scripts.seed
```

### ログ確認

```powershell
docker compose logs backend
```

直近のログだけ確認する場合は、次のコマンドを使います。

```powershell
docker compose logs backend --tail=120
```

### 起動状態の確認

```powershell
docker compose ps
```

`backend`、`db`、`mailhog` が起動していれば、基本的な動作確認を始められます。

> 本番環境の起動・更新・SMTP 切替（`mailmode.sh`）などサーバー運用の手順は `../INFRASTRUCTURE.md` / `../HANDOFF.md` を参照してください。
