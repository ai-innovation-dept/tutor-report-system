# クローズ／強制クローズ ガイド（やさしい解説・具体例つき）

> 2システム共通の解説。月次の報告書が承認フローの途中で詰まって「完了」できないときに、運営／管理スタッフが**理由を付けて手動で終了させる**機能について、「いつ・誰が・どう操作するか」を具体例で説明します。
> 正式な仕様・項目は次を参照: 既存＝`イスト勤怠レポート for 代々木進学会/SPECIFICATION.md` §8「未処理報告クローズ機能」、新＝`イスト勤怠レポート for EMPS/OPERATION_MANUAL.md`・`SPECIFICATION.md`、データ項目＝`DATA_MODEL.md`。索引＝`README.md`。
> 最終更新: 2026-06-22

---

## まず一言で言うと

報告書は本来 **「作成 → 承認 → 完了」** と進みます。しかし途中で誰かが止まってしまい、**いつまでも「完了」できずに残り続ける報告書**が出ることがあります。それを運営スタッフが **「理由を書いて、手動で打ち切り終了にする」** のが **クローズ／強制クローズ** です。

- 「承認して完了」ではなく、**「正常に終われないので、運営判断で打ち切って終わらせる」**操作。だから「**強制**」と付きます。
- **データは消えません**。履歴に「クローズした理由・実行者・日時」が残ります。
- 目的：**未処理のまま居座る報告書を片付け**、運営の確認リストをきれいに保つこと。

---

## 具体例①：イスト勤怠レポート for 代々木進学会（指導実績報告）

**登場人物**：講師の山田先生／生徒・田中さんの保護者／運営スタッフ

1. 山田先生が **「4月分の指導実績報告書」** を作り、田中さんの保護者へ「承認してください」と送る。
2. ところが保護者が**ずっと承認ボタンを押さない**（多忙、あるいは退会して連絡が取れない）。
3. 報告書は「**保護者の承認待ち**」のまま、4月 → 5月 → 6月…と**残り続ける**。
4. 5月になると、システムが「先月(4月)以前なのに終わっていない＝**未処理**」と判断し、運営画面に **「未処理 ◯件」** とバナーを表示。放置が続くと **7日・14日・30日でリマインド（エスカレーション）メール**を送る。
5. 運営（**受付・再鑑・管理者・管理責任者**のいずれでも可）が「もう保護者と連絡が取れない。このまま残せない」と判断。
6. **「未処理報告一覧」画面**を開き、その報告書に **「保護者が退会し連絡不能のためクローズ」と理由を入力してクローズ**。
7. → 報告書は「**クローズ済（終了）**」になり、未処理リストから消える。記録は残る。

> ポイント：**正常に承認が終わらず、月をまたいで居座る報告書を、運営が理由付きで片付ける**機能。

## 具体例②：イスト勤怠レポート for EMPS（業務連絡表）

**登場人物**：派遣講師の鈴木先生／派遣先の学校担当者／事務・営業

1. 鈴木先生が **「4月分の業務連絡表（勤務実績）」** を提出。本来は **学校が確認 → 事務 → 営業が承認＝完了** と回る。
2. ところが**学校の担当者が確認ボタンを押さない**まま放置（年度替わりで担当者が交代した、など）。
3. 報告書は「**学校の確認待ち**」のまま、4月 → 5月と**滞留**する。
4. 5月になると、**進行中の当月(5月)より前なのに終わっていない**案件として滞留リストに出る。
5. **営業・事務・経理・管理責任者**の誰かが、自分の**キュー（パイプライン）画面**で「学校都合で確認が取れない。締めよう」と判断。
6. その行の **「強制クローズ」** を押し、**「学校担当者交代により確認不能。営業確認のうえクローズ」と理由を入力**。
7. → 報告書は「**クローズ（終了）**」になり滞留が解消。講師・学校へ通知が飛ぶ。

> 安全装置：**今まさに作業中の当月分はクローズできません**（誤って締めないため）。

## 「普通の完了」と「クローズ」の違い（ここが肝心）

| | 正常な終わり方 | クローズ／強制クローズ |
|---|---|---|
| どうやって終わる | 保護者や営業が**承認**して完了 | 承認を**待たずに運営が打ち切り**で終了 |
| いつ使う | 通常運用 | 連絡不能・放置などで**承認が進まず滞留**したとき |
| 必須事項 | — | **クローズ理由が必須**（後から経緯が分かるように） |
| データ | 残る | 残る（**削除はしない**） |

## 2システムの違い（入口だけが主な差）

| | 代々木進学会（既存） | EMPS（新） |
|---|---|---|
| 名称 | 未処理報告クローズ | 強制クローズ |
| 入口 | 専用「未処理報告一覧」画面（`/admin/stale-reports`） | 各キュー（パイプライン）画面の行から |
| 対象 | 先月以前 × 未完了 | 当月より前 × 未完了（**当月は不可**） |
| 実行できる人 | 受付／再鑑／管理者／管理責任者 | 営業／事務／経理／管理責任者 |
| 理由 | 必須 | 必須 |
| 通知 | 7／14／30日でエスカレーション通知 | クローズ時に関係者へ通知 |

## まとめ

両システムとも、**月次の報告書が承認フローの途中で詰まって完了できないときに、運営／管理スタッフが理由を添えて手動で終了させる「お片付け・打ち切り」機能**です。通常の「承認による完了」とは別物で、あくまで**例外対応**として使います。
