# 家庭教師 指導実績報告システム 操作手順書

更新日: 2026年5月22日

## 1. システム概要

このシステムは、講師が指導実績を登録し、保護者が月ごとに確認し、運営が受付、再鑑、最終承認まで進めるためのものです。

| 登場人物 | 役割 | 主な操作 |
|---|---|---|
| 講師 | 指導後に報告書を作成し、月ごとに承認依頼と運営提出を行います。 | 報告書作成、修正、削除、保護者への承認依頼、運営提出 |
| 保護者 | 講師から依頼された報告書を月ごとに確認します。 | 月次確認、まとめて承認、差戻し |
| 運営 | 提出された報告書を講師、生徒、月の単位で確認します。 | 受付、再鑑、最終承認、差戻し、ユーザー管理 |

運営には3つの役割があります。

| 運営ロール | 役割 |
|---|---|
| 運営受付 | 講師から運営へ提出された報告書を受け付けます。 |
| 運営再鑑 | 受付済みの報告書を再確認します。 |
| 運営管理者 | 最終承認、ユーザー作成、講師と保護者の紐付けを行います。 |

## 2. ログインログアウト

### ログイン手順

1. ブラウザで `http://localhost:8000/login` を開きます。
2. メールアドレスを入力します。
3. パスワードを入力します。
4. `Login` ボタンを押します。
5. ログインに成功すると、自分の役割に合った画面へ移動します。

### ログアウト手順

1. 画面上部の `Logout` ボタンを押します。
2. ログイン画面に戻ればログアウト完了です。

## 4. ロール別 操作手順

### 講師（tutor）

#### 報告書の新規作成手順

1. 講師アカウントでログインします。
2. `http://localhost:8000/tutor/reports` を開きます。
3. 右側の `簡易作成` フォームに入力します。
4. 生徒を選びます。
5. 指導日、開始時刻、終了時刻、休憩等の時間、科目、指導内容を入力します。
6. 指導時間数が自動で計算されることを確認します。
7. `保存` ボタンを押します。
8. 一覧に報告書が追加されれば作成完了です。

#### 新しい生徒を追加して報告書を作成する手順

1. `http://localhost:8000/tutor/reports` を開きます。
2. 生徒プルダウンで `＋ 新しい生徒を追加` を選びます。
3. 表示された生徒名入力欄に生徒名を入力します。
4. 指導日、時刻、科目、指導内容を入力します。
5. `保存` ボタンを押します。
6. 生徒が登録され、その生徒の報告書が作成されます。

新しく追加した生徒は、最初は保護者未連携です。保護者と紐付ける場合は、運営管理者が後から紐付けを行います。

#### 報告書の確認、編集、削除

1. `http://localhost:8000/tutor/reports` を開きます。
2. 月選択プルダウンで確認したい月を選びます。
3. 生徒プルダウンで確認したい生徒を選びます。
4. 下書きの報告書は `編集` と `削除` ができます。
5. 差戻し中の報告書は `編集` ができます。
6. 過去月や提出済みの報告書は、基本的に参照のみです。

#### 保護者へ月次承認依頼する手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. 対象月のカードを確認します。
3. その月の報告書がすべて下書きから進められる状態であれば、`保護者へまとめて承認依頼を送る` ボタンが表示されます。
4. ボタンを押します。
5. 保護者側の状態が `承認待ち` になれば依頼完了です。

#### 差戻されたときの修正再依頼手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. 対象月に `差戻しがあります` と表示されていることを確認します。
3. `http://localhost:8000/tutor/reports` を開きます。
4. 差戻し理由を確認します。
5. 対象の報告書を編集します。
6. 修正後、再度 `http://localhost:8000/tutor/approval` から保護者へ承認依頼します。

#### 運営へ月次提出する手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. 対象月の保護者ステップが `承認済み` になっていることを確認します。
3. `運営へまとめて提出する` ボタンを押します。
4. 運営ステップが `提出済み` になれば運営への提出完了です。

### 保護者（parent）

#### 承認依頼を確認する手順

1. 保護者アカウントでログインします。
2. `http://localhost:8000/parent/reports` を開きます。
3. 画面上部の月選択プルダウンで確認したい月を選びます。
4. 選択した月の報告書だけが表示されます。
5. 指導日、在室時間、指導時間数、科目、指導内容を確認します。

未来月は月選択に表示されません。過去月は参照のみで、承認や差戻しはできません。

#### 月次承認する手順

1. `http://localhost:8000/parent/reports` を開きます。
2. 月選択で当月を選びます。
3. 報告書の内容を確認します。
4. 問題がなければ `5月分を承認する` のような承認ボタンを押します。
5. 確認ダイアログで内容を確認し、承認します。
6. 表示が `承認済み` に切り替われば完了です。

#### 差戻し手順

1. `http://localhost:8000/parent/reports` を開きます。
2. 月選択で当月を選びます。
3. 内容に修正が必要な場合は `差戻しにする` ボタンを押します。
4. 差戻し理由を入力します。
5. 差戻しを実行します。
6. 講師側に差戻し理由が表示されます。

差戻し理由は、講師が修正内容を理解できるように具体的に書いてください。

### 運営受付（admin_receiver）

#### 受付待ち一覧の確認手順

1. 運営受付アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/receive` を開きます。
3. 月選択プルダウンで対象月を選びます。
4. 講師選択プルダウンで対象講師を選びます。全員を確認する場合は `すべての講師` を選びます。
5. 生徒ごとのカードを確認します。
6. カードのステータスが `受付待ち` のものが受付対象です。

#### 受付処理の手順

1. 受付待ちの生徒カードを確認します。
2. 必要に応じて `指導記録を見る` を押し、明細を確認します。
3. 問題がなければ `5月分を受付する` のようなボタンを押します。
4. 確認ダイアログで内容を確認し、受付します。
5. カードのステータスが `再鑑待ち` になれば受付完了です。

#### 差戻し手順

1. 受付待ちの生徒カードを確認します。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. カードのステータスが `差戻し中` になれば差戻し完了です。

### 運営再鑑（admin_reviewer）

#### 再鑑待ち一覧の確認手順

1. 運営再鑑アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/review` を開きます。
3. 月選択と講師選択を使って対象を絞り込みます。
4. 生徒ごとのカードを確認します。
5. カードのステータスが `再鑑待ち` のものが再鑑対象です。

#### 再鑑処理の手順

1. 再鑑待ちの生徒カードを確認します。
2. `指導記録を見る` を押し、明細を確認します。
3. 問題がなければ `5月分を再鑑する` のようなボタンを押します。
4. 確認ダイアログで内容を確認し、再鑑します。
5. カードのステータスが `最終承認待ち` になれば再鑑完了です。

#### 差戻し手順

1. 再鑑待ちの生徒カードを確認します。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. カードのステータスが `差戻し中` になれば差戻し完了です。

### 運営管理者（admin_master）

#### 最終承認手順

1. 運営管理者アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/approve` を開きます。
3. 月選択と講師選択を使って対象を絞り込みます。
4. 生徒ごとのカードを確認します。
5. カードのステータスが `最終承認待ち` のものが最終承認対象です。
6. `指導記録を見る` を押し、明細を確認します。
7. 問題がなければ `5月分を最終承認する` のようなボタンを押します。
8. カードのステータスが `最終承認済み` になれば完了です。

運営管理者は、受付待ちキューでは受付、再鑑待ちキューでは再鑑も行えます。

#### 差戻し手順

1. 対象の生徒カードを確認します。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. カードのステータスが `差戻し中` になれば差戻し完了です。

#### ユーザー作成手順

1. 運営管理者アカウントでログインします。
2. `http://localhost:8000/admin/users` を開きます。
3. 新規ユーザー作成欄で、メールアドレス、名前、ロールを入力します。
4. 作成ボタンを押します。
5. 初期パスワードが表示された場合は、対象者へ安全な方法で伝えます。

現在の画面は簡易表示です。詳細な登録確認は `http://localhost:8000/docs` の `/api/users` からも行えます。

#### 講師と保護者の紐付け作成手順

1. 運営管理者アカウントでログインします。
2. `http://localhost:8000/admin/assignments` を開きます。
3. 講師、保護者、生徒名を入力します。
4. 作成または更新を行います。
5. 紐付けが完了すると、講師はその生徒の報告書を作成でき、保護者はその報告書を確認できます。

講師がフォームから追加した新しい生徒は、最初は保護者未連携です。必要に応じて管理者が後から保護者を設定します。

## 5. リマインド通知について

リマインド通知は、月末が近づいたときに自動で作成されます。標準設定では、月末の3日前から月末当日までが対象です。

| 状態 | 通知先 | 内容 |
|---|---|---|
| 保護者承認待ち | 保護者 | 未承認の報告書があることを知らせます。 |
| 下書きのまま | 講師 | まだ提出されていない報告書があることを知らせます。 |
| 差戻し後、未再提出 | 講師 | 修正と再提出が必要な報告書があることを知らせます。 |
| 保護者承認済みだが運営未提出 | 講師 | 運営への提出が必要な報告書があることを知らせます。 |

開発環境では、送信されたメールは `http://localhost:8025` の MailHog で確認できます。

## 6. よくある操作ミスと対処法

### 誤って提出してしまった場合

提出後は、講師側で自由に削除や修正ができない状態になります。内容に誤りがある場合は、保護者または運営に連絡し、差戻しを依頼してください。

差戻し後、講師は内容を修正して再提出できます。

### 差戻しコメントを入力し忘れた場合

差戻しには理由コメントが必要です。コメントを入力しないと、差戻しは完了しません。

コメント未入力でエラーになった場合は、もう一度差戻し操作を行い、理由を入力してから送信してください。

例として、次のように具体的に書くと相手が対応しやすくなります。

「指導時間が実際と異なるため修正してください」

「科目名が未入力のため追記してください」

### 月を間違えて見ている場合

報告書が見つからない場合は、画面上部の月選択を確認してください。保護者画面と運営画面は、選択した1か月分だけを表示します。

### 過去月でボタンが表示されない場合

過去月は参照のみです。承認、差戻し、受付、再鑑、最終承認のボタンは表示されません。

## 7. 管理者向け：起動停止手順

以下のコマンドは、`C:\PerformanceReportingSystem\tutor-report-system` フォルダで実行してください。

### 起動

```powershell
docker compose up -d --build
```

### 停止

```powershell
docker compose down
```

### 再起動

```powershell
docker compose restart backend
```

### マイグレーション適用

```powershell
docker compose exec backend alembic upgrade head
```

### シードデータ投入

```powershell
docker compose exec backend python -m app.scripts.seed
```

### ログ確認

```powershell
docker compose logs backend
```

直近のログだけ確認する場合は、次のコマンドを使います。

```powershell
docker compose logs backend --tail=120
```

### 起動状態の確認

```powershell
docker compose ps
```

`backend`、`db`、`mailhog` が起動していれば、基本的な動作確認を始められます。
