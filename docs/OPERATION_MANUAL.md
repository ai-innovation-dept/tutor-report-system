更新日: 2026年6月5日

> 本書は **2つのシステム** の操作手順を扱います。
> - **第 I 部**: 旧システム＝**家庭教師 指導実績報告**（`http://localhost:8000`。講師→保護者→運営）

# 第 I 部：旧システム（家庭教師 指導実績報告 / port 8000）

## 1. システム概要

このシステムは、講師が指導実績を登録し、保護者が月ごとに確認し、運営が受付、再鑑、最終承認まで進めるためのものです。

| 登場人物 | 役割 | 主な操作 |
|---|---|---|
| 講師 | 指導後に報告書を作成し、月ごとに保護者へ承認依頼します。 | 報告書作成、修正、削除、保護者への承認依頼 |
| 保護者 | 講師から依頼された報告書を月ごとに確認します。承認すると運営へ自動提出されます。 | 月次確認、まとめて承認、差戻し |
| 運営 | 提出された報告書を講師、生徒、月の単位で確認します。 | 受付、再鑑、最終承認、差戻し、ユーザー管理 |

運営には3つの役割があります。

| 運営ロール | 役割 |
|---|---|
| 運営受付 | 講師から運営へ提出された報告書を受け付けます。 |
| 運営再鑑 | 受付済みの報告書を再確認します。 |
| 運営管理者 | 最終承認、ユーザー作成、講師と保護者の紐付けを行います。 |

## 2. ログインログアウト

### ログイン手順

1. ブラウザで `http://localhost:8000/login` を開きます。
2. メールアドレスを入力します。
3. パスワードを入力します。
4. `ログイン` ボタンを押します。
5. ログインに成功すると、自分の役割に合った画面へ移動します。

### ログアウト手順

1. 画面上部の `Logout` ボタンを押します。
2. ログイン画面に戻ればログアウト完了です。

### パスワードを忘れた場合

1. ログイン画面を開きます。
2. `パスワードをお忘れの方はこちら` を押します。
3. 登録済みのメールアドレスを入力します。
4. `リセットメールを送る` を押します。
5. `メールを送信しました。ご確認ください。` と表示されたら、メールを確認します。
6. メール本文のパスワードリセットURLを開きます。
7. 新しいパスワードと確認用パスワードを入力します。
8. `パスワードを変更する` を押します。
9. `パスワードを変更しました` と表示され、3秒後にログイン画面へ移動します。
10. 新しいパスワードでログインします。

リセットURLの有効期限は1時間です。期限切れまたは使用済みの場合は、もう一度リセットメールを送信してください。開発環境ではメールは MailHog（`http://localhost:8025`）で確認します。

## 4. ロール別 操作手順

### 講師（tutor）

#### 報告書の新規作成手順

1. 講師アカウントでログインします。
2. `http://localhost:8000/tutor/reports` を開きます。
3. 右側の `簡易作成` フォームに入力します。
4. 生徒を選びます。
5. 指導日、開始時刻、終了時刻、休憩等の時間、科目、指導内容を入力します。
6. 指導時間数が自動で計算されることを確認します。
7. `保存` ボタンを押します。
8. 一覧に報告書が追加されれば作成完了です。

#### 新しい生徒を追加して報告書を作成する手順

1. `http://localhost:8000/tutor/reports` を開きます。
2. 生徒プルダウンで `＋ 新しい生徒を追加` を選びます。
3. 表示された生徒名入力欄に生徒名を入力します。
4. 指導日、時刻、科目、指導内容を入力します。
5. `保存` ボタンを押します。
6. 生徒が登録され、その生徒の報告書が作成されます。

新しく追加した生徒は、最初は保護者未連携です。保護者と紐付ける場合は、運営管理者が後から紐付けを行います。

#### 報告書の確認、編集、削除

1. `http://localhost:8000/tutor/reports` を開きます。
2. 月選択プルダウンで確認したい月を選びます。
3. 生徒プルダウンで確認したい生徒を選びます。
4. 下書きの報告書は `編集` と `削除` ができます。
5. 差戻し中の報告書は `編集` ができます。
6. 過去月や提出済みの報告書は、基本的に参照のみです。

#### 保護者へ月次承認依頼する手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. 対象月のカードを確認します。
3. その月の報告書がすべて下書きから進められる状態であれば、`保護者へまとめて承認依頼を送る` ボタンが表示されます。
4. ボタンを押します。
5. 保護者側の状態が `承認待ち` になれば依頼完了です。

#### 差戻されたときの修正再依頼手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. 対象月に `差戻しがあります` と表示されていることを確認します。
3. `http://localhost:8000/tutor/reports` を開きます。
4. 差戻し理由を確認します。
5. 対象の報告書を編集します。
6. 修正後、再度 `http://localhost:8000/tutor/approval` から保護者へ承認依頼します。

#### 講師のエクスポート手順

1. `http://localhost:8000/tutor/approval` を開きます。
2. エクスポートしたい月を選びます。
3. 生徒別に出力する場合は、最終承認済みカードの `Excelでダウンロード` または `CSVでダウンロード` を押します。
4. 対象月の全生徒をまとめて出力する場合は、上部のダウンロードボタンを押します。

講師の一括エクスポートは、その月の対象生徒がすべて最終承認済みの場合のみ有効です。

#### 運営提出について

現在の実装では、保護者が月次承認するとシステムが自動で運営へ提出します。講師が保護者承認後に別途 `運営へ提出` を押す運用ではありません。

`/tutor/reports` には個別・一括の運営提出 API を呼ぶ古い操作導線が一部残っていますが、通常運用では `/tutor/approval` で保護者依頼までを行い、その後は保護者承認を待ちます。

### 保護者（parent）

#### 承認依頼を確認する手順

1. 保護者アカウントでログインします。
2. `http://localhost:8000/parent/reports` を開きます。
3. 画面上部の月選択プルダウンで確認したい月を選びます。
4. 選択した月の報告書だけが表示されます。
5. 指導日、在室時間、指導時間数、科目、指導内容を確認します。

未来月は月選択に表示されません。過去月は参照のみで、承認や差戻しはできません。

#### 月次承認する手順

1. `http://localhost:8000/parent/reports` を開きます。
2. 月選択で当月を選びます。
3. 報告書の内容を確認します。
4. 問題がなければ `5月分を承認する` のような承認ボタンを押します。
5. 確認ダイアログで内容を確認し、承認します。
6. 表示が `承認済み` または運営側の `受付待ち` に進めば完了です。承認と同時に運営へ自動提出されます。

#### 月次差戻し手順

1. `http://localhost:8000/parent/reports` を開きます。
2. 月選択で当月を選びます。
3. 内容に修正が必要な場合は `5月分を差戻す` のような月次差戻しボタンを押します。
4. 差戻し理由を入力します。
5. 差戻しを実行します。
6. 対象月の承認待ち報告書がまとめて講師へ差戻され、講師側に差戻し理由が表示されます。

差戻し理由は、講師が修正内容を理解できるように具体的に書いてください。

#### 保護者のエクスポート手順

1. `http://localhost:8000/parent/reports` を開きます。
2. 月と生徒を選びます。
3. 保護者承認待ちの報告書が残っていない月では、`Excelでダウンロード` または `CSVでダウンロード` が表示されます。
4. 全生徒をまとめて出力する場合は、上部の一括ダウンロードボタンを押します。

### 運営受付（admin_receiver）

#### 受付待ち一覧の確認手順

1. 運営受付アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/receive` を開きます。
3. 月選択プルダウンで対象月を選びます。
4. 講師選択プルダウンで対象講師を選びます。全員を確認する場合は `すべての講師` を選びます。
5. 生徒ごとのカードを確認します。
6. カードのステータスが `受付待ち` のものが受付対象です。

#### 受付処理の手順

1. 受付待ちの生徒カードを確認します。
2. 必要に応じて `指導記録を見る` を押し、明細を確認します。
3. 問題がなければ `5月分を受付する` のようなボタンを押します。
4. 確認ダイアログで内容を確認し、受付します。
5. カードのステータスが `再鑑待ち` になれば受付完了です。

#### 月次差戻し手順

1. 受付待ちの生徒カードを確認します。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. 対象生徒・対象月の報告書がまとめて `差戻し中` になれば差戻し完了です。

#### 運営のエクスポート手順

1. `http://localhost:8000/admin/dashboard` を開きます。
2. 月選択と講師選択で対象を絞り込みます。
3. 生徒カードの `Excelでダウンロード` または `CSVでダウンロード` を押すと、生徒別に出力できます。
4. 上部の月別ダウンロードボタンを押すと、対象月の全体を一括出力できます。

運営ロールはステータスに関係なく、存在する報告書を出力できます。

### 運営再鑑（admin_reviewer）

#### 再鑑待ち一覧の確認手順

1. 運営再鑑アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/review` を開きます。
3. 月選択と講師選択を使って対象を絞り込みます。
4. 生徒ごとのカードを確認します。
5. カードのステータスが `再鑑待ち` のものが再鑑対象です。

#### 再鑑処理の手順

1. 再鑑待ちの生徒カードを確認します。
2. `指導記録を見る` を押し、明細を確認します。
3. 問題がなければ `5月分を再鑑する` のようなボタンを押します。
4. 確認ダイアログで内容を確認し、再鑑します。
5. カードのステータスが `最終承認待ち` になれば再鑑完了です。

#### 月次差戻し手順

1. 再鑑待ちの生徒カードを確認します。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. 対象生徒・対象月の報告書がまとめて `差戻し中` になれば差戻し完了です。

### 運営管理者（admin_master）

#### 最終承認手順

1. 運営管理者アカウントでログインします。
2. `http://localhost:8000/admin/dashboard` または `http://localhost:8000/admin/queue/approve` を開きます。
3. 月選択と講師選択を使って対象を絞り込みます。
4. 生徒ごとのカードを確認します。
5. カードのステータスが `最終承認待ち` のものが最終承認対象です。
6. `指導記録を見る` を押し、明細を確認します。
7. 問題がなければ `5月分を最終承認する` のようなボタンを押します。
8. カードのステータスが `最終承認済み` になれば完了です。

運営管理者は、受付待ちキューでは受付、再鑑待ちキューでは再鑑も行えます。

#### 月次差戻し手順

1. 対象の生徒カードを確認します。
2. 内容に不備がある場合は `差戻し` ボタンを押します。
3. 差戻し理由を入力します。
4. 対象生徒・対象月の報告書がまとめて `差戻し中` になれば差戻し完了です。

#### ユーザー追加手順（招待方式）

1. 運営管理者アカウントでログインします。
2. `http://localhost:8000/admin/users` を開きます。
3. 新規ユーザー登録欄でロールを選びます。
4. 講師・運営スタッフの場合は氏名とメールアドレスを入力します。
5. 保護者の場合はメールアドレス、担当講師、生徒名を入力します。氏名は登録時に自動設定されます。
6. `招待メールを送る` を押します。
7. 登録済みユーザー一覧に「未登録」として表示され（招待済み一覧は廃止し、登録済みユーザー一覧に統合）、招待先ユーザーがメールのリンクからパスワードを設定すると登録完了です。

講師番号は講師招待時に自動採番されます。招待リンクの有効期限は72時間です。未登録の招待は一覧の `再送` または `取消` ができます。登録済みユーザーは詳細から有効化・無効化できます。

#### 保護者承認スキップを設定する手順

旧「システム設定」画面は廃止しました。保護者承認のスキップはユーザー管理画面で設定します。

1. 運営管理者アカウントでログインします。
2. `http://localhost:8000/admin/users` を開きます。
3. 対象の保護者ユーザーの `詳細` を開きます。
4. 「保護者承認設定」の `保護者承認をスキップする` をオンにします。
5. オンにすると、その保護者の担当報告書は保護者承認を省略して直接運営へ提出されるようになります。

担当紐付け（既存生徒への講師追加）は、講師の生徒管理画面と保護者招待（`/admin/users`）から作成します。リマインドは講師が承認管理画面（承認依頼時）で設定します。

## 5. リマインド通知について

リマインド通知は、月末が近づいたときに自動で作成されます。標準設定では、月末の3日前から月末当日までが対象です。

| 状態 | 通知先 | 内容 |
|---|---|---|
| 保護者承認待ち | 保護者 | 未承認の報告書があることを知らせます。 |
| 下書きのまま | 講師 | まだ提出されていない報告書があることを知らせます。 |
| 差戻し後、未再提出 | 講師 | 修正と再提出が必要な報告書があることを知らせます。 |
| 保護者承認済みだが運営未提出 | 講師 | 通常は保護者承認時に自動提出されるため短時間のみ発生します。残っている場合は運営への提出が必要な報告書があることを知らせます。 |

開発環境では、送信されたメールは `http://localhost:8025` の MailHog で確認できます。

## 6. よくある操作ミスと対処法

### 誤って提出してしまった場合

提出後は、講師側で自由に削除や修正ができない状態になります。内容に誤りがある場合は、保護者または運営に連絡し、差戻しを依頼してください。

差戻し後、講師は内容を修正して再提出できます。

### 差戻しコメントを入力し忘れた場合

差戻しには理由コメントが必要です。コメントを入力しないと、差戻しは完了しません。

コメント未入力でエラーになった場合は、もう一度差戻し操作を行い、理由を入力してから送信してください。

例として、次のように具体的に書くと相手が対応しやすくなります。

「指導時間が実際と異なるため修正してください」

「科目名が未入力のため追記してください」

### 月を間違えて見ている場合

報告書が見つからない場合は、画面上部の月選択を確認してください。保護者画面と運営画面は、選択した1か月分だけを表示します。

### 過去月でボタンが表示されない場合

過去月は参照のみです。承認、差戻し、受付、再鑑、最終承認のボタンは表示されません。

# 第 II 部：新システム（業務連絡表 / port 8001）

新システムは、学校へ派遣された講師が月次の「業務連絡表」を作成し、**学校 → 事務 → 営業 → 経理** の順で承認を得るためのシステムです。URL は `http://localhost:8001` です。

## 8. 新システムの概要

| 登場人物 | 役割 | 主な操作 |
|---|---|---|
| 講師（tutor） | 派遣先（学校）ごとに業務連絡表を作成し提出します。 | 報告書の作成・編集・削除・提出・再提出 |
| 学校（school） | 講師の業務連絡表を確認します。 | 承認、差戻し（講師へ） |
| 事務（office） | 学校承認後の確認、営業・経理からの差戻し対応を行います。 | 承認（営業へ前進）、差戻し（講師へ） |
| 営業（sales） | 事務確認後の確認を行います。 | 承認、差戻し（事務へ） |
| 経理（admin_master） | 最終承認、契約管理、ユーザー管理を行います。 | 最終承認、完了後の差戻し、契約管理、ユーザー管理（学校承認スキップ設定を含む） |

承認の流れ: `下書き → 学校承認待ち → 事務確認待ち → 営業確認待ち → 経理確認待ち → 完了`。学校承認が不要な学校（ユーザー管理で学校承認スキップを設定）では、提出時に学校確認をスキップして事務確認へ進みます。

## 9. ログインとデモアカウント

### ログイン手順

1. ブラウザで `http://localhost:8001/login` を開きます。
2. メールアドレスとパスワードを入力し、`ログイン` を押します。
3. 複数の役割を持つアカウントは、ログイン後に役割選択画面が表示されます。使用する役割を選びます。
4. 役割に応じた初期画面へ移動します。

パスワードを忘れた場合の手順は旧システムと同じです（リセットメール → リンク → 新パスワード設定）。開発環境のメールは MailHog（`http://localhost:8025`）で確認します。

### デモアカウント

パスワードはすべて `Passw0rd!` です。学校・営業・事務のアカウントは `docker compose exec new_backend python -m app.scripts.create_test_users` で作成します。講師・経理は旧システムのシードアカウント（`tutor1@example.com` / `master1@example.com`）を共有します。

|---|---|---|
| 講師 | tutor1@example.com | 講師 一郎（共有） |
| 学校 | school1@example.com | 椅子戸学園 |

## 10. 講師の操作手順

### 業務連絡表を作成する手順

1. 講師アカウントでログインし、`http://localhost:8001/tutor/reports` を開きます。
2. 画面上部の月選択で対象月を確認します（作成できるのは当月分のみです）。
3. `派遣先事業所の名称・組織単位` のプルダウンから学校を選びます。
   - このプルダウンには、**経理の契約管理に登録された自分の契約校のみ** が表示されます。表示されない場合は経理へ契約登録を依頼してください。
4. 学校を選ぶと、その学校の契約内容（委託業務・採点の有無など）に応じてフォームの列が切り替わります。
5. 明細行に、日付・業務開始/終了時間・担当時限・各委託業務の分数・休憩・往復交通費・内容を入力します。
   - 終了時間が開始時間より早い場合は保存できません（警告が表示されます）。
   - 採点欄が有効な契約では、採点の「回」「分」を1セルに併記入力します。
6. `保存` を押します。一覧に業務連絡表が追加されれば作成完了です。

複数の学校を担当する場合は、`＋別の学校の報告書を作成` ボタンや横タブで学校を切り替えて、学校ごとに並行して作成できます。

### 確認・編集・削除

1. `http://localhost:8001/tutor/reports` を開きます。
2. 横タブで対象の学校（報告書）を選びます。
3. 下書き・差戻し中の報告書は編集できます。
4. 下書き・差戻し中の報告書は `報告を削除` で削除できます（本人分のみ）。
5. 提出済み・過去月の報告書は参照のみです。

### 提出・再提出する手順

1. `http://localhost:8001/tutor/approval`（承認管理）を開きます。
2. 対象の学校・月のカードを確認します。
3. 内容に問題がなければ提出（承認依頼）を行います。状態が `学校承認待ち`（または学校承認スキップが設定された学校では `事務確認待ち`）に進みます。承認依頼時にリマインド（送付間隔・回数）も設定できます。
4. 差戻しがある場合はカードに差戻し理由が表示されます。`報告書一覧` で内容を修正し、承認管理から再提出します。

## 11. 学校の操作手順

1. 学校アカウントでログインし、`http://localhost:8001/school/approval` を開きます。
2. 講師×月ごとのカード（「YYYY年M月分　学校名：〇〇　講師名：●●」）が表示されます。
3. `報告書を確認` を押すと、別ウィンドウで業務連絡表を参照できます。
4. 内容に問題がなければ `承認する` を押します。状態が事務確認待ちへ進みます。
5. 修正が必要な場合は `差戻し` を押し、理由を入力します。報告書は講師へ差し戻されます。

## 12. 事務・営業・経理の操作手順

事務（`/office/queue`）・営業（`/sales/queue`）・経理（`/finance/queue`）の3画面は同じ構造で、`タスク`（自分の対応待ち）と `パイプライン`（全体の進捗列）で構成されます。

### 共通操作

1. 各ロールのアカウントでログインし、対応する画面を開きます。
2. `タスク` で自分の対応待ち（自ロールの確認待ち）カードを確認します。
3. `報告書を確認` を押して内容を参照します。
4. 承認または差戻しを行います。

### 各ロールの承認・差戻し

| ロール | 画面 | 承認の効果 | 差戻し先 |
|---|---|---|---|
| 事務 | `/office/queue` | 営業確認待ちへ前進 | 講師（差戻し時） |
| 営業 | `/sales/queue` | 経理確認待ちへ前進 | 事務 |
| 経理 | `/finance/queue` | 完了（最終承認） | 事務 |

- 営業・経理からの差戻しは **事務** に届きます（`事務へ差戻し中`）。事務は「承認＝営業へ前進」または「差戻し＝講師へ」を選びます。
- 経理は完了（最終承認済み）後でも、修正が必要な場合は差戻しできます（事務へ戻ります）。
- パイプライン列の `強制クローズ` で、滞留した報告を終了させることができます（理由必須）。

## 13. 契約管理（経理）

経理（管理者）は `http://localhost:8001/admin/contracts` で、講師×学校ごとの契約を管理します。契約はその講師の業務連絡表フォーム（委託業務・採点の列）の元になります。

### 契約を1件ずつ登録する手順

1. 経理アカウントでログインし、`/admin/contracts` を開きます。
2. `新規登録` で登録ドロワーを開きます。
3. 講師・学校・お客様ID・弊社担当・契約期間・月固定分・週コマ数などを入力します。
4. 委託業務①〜⑤に、業務名・委託業務ID・個別契約IDを入力します（委託業務①は必須）。
5. 採点を報告書に出す場合は `採点を追加する` をチェックし、採点の委託業務ID・個別契約IDを入力します。
6. 保存すると、対応する紐付け（assignment）が自動で作成・更新されます。

### CSV で一括登録する手順

1. `/admin/contracts` 上部の `CSVテンプレート` を押してテンプレート（記入例付き）をダウンロードします。
2. テンプレートに沿って契約を記入します。
   - 講師は **講師番号**、学校は **学校名** で指定します。
   - 記入例・コメント行（講師番号が空、または先頭が `#`）と空行は取り込まれません。
3. `CSVインポート` でファイルを選択してアップロードします（文字コードは UTF-8 / Shift-JIS を自動判定）。
4. 結果モーダルで取り込み件数を確認します。
   - 1件でもエラーがある場合は、行番号付きのエラー一覧が表示され、**全件が中止**されます。エラーを直して再アップロードしてください。
   - 同じ（講師×学校）の契約が既にある場合は上書き（更新）されます。

## 14. 新システムの通知・リマインド

## 15. 新システムの起動・停止（管理者向け）

新システムは旧システムと同じ Docker Compose で同時に起動します。

```bash
# 起動（旧 backend:8000 / 新 new_backend:8001 / db / mailhog がまとめて起動）
docker compose up -d --build

# 新システムのみログ確認
docker compose logs new_backend -f

# 新システムのマイグレーション適用（通常は起動時に自動実行）
docker compose exec new_backend alembic upgrade head

# 学校・営業・事務のデモユーザー作成
docker compose exec new_backend python -m app.scripts.create_test_users
```

新システムのマイグレーションは旧システムと分離して管理されます（バージョンテーブル `work_alembic_version`）。新システムのコンテナは起動時に `alembic upgrade head` を自動実行します。
