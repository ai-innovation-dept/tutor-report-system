# イスト勤怠レポート for EMPS — 操作手順書（OPERATION MANUAL）

> 旧称: 業務連絡表システム（新システム）。`new_backend/`・ポート8001。
> 本書は2システム構成のうち「イスト勤怠レポート for EMPS」の操作手順書です。
> 共通情報: データモデル `../DATA_MODEL.md` / インフラ `../INFRASTRUCTURE.md` / 引継ぎ `../HANDOFF.md` / 索引 `../README.md`
> 最終更新: 2026-06-22

イスト勤怠レポート for EMPS（旧称: 業務連絡表システム）は、学校へ派遣された講師が月次の「業務連絡表」を作成し、**学校 → 事務 → 営業** の順で承認を得るためのシステムです。URL は `http://localhost:8001` です。

> 用語: 本書の「業務連絡表」は講師が作成する報告書そのものを指します。システム名は「イスト勤怠レポート for EMPS」です。

## 1. システムの概要

| 登場人物 | 役割 | 主な操作 |
|---|---|---|
| 講師（tutor） | 派遣先（学校）ごとに業務連絡表を作成し提出します。 | 報告書の作成・編集・削除・提出・再提出 |
| 学校（school） | 講師の業務連絡表を確認します。 | 承認、差戻し（講師へ） |
| 事務（office） | 学校承認後の確認、報告書の修正、営業からの差戻し対応を行います。 | 承認（営業へ前進）、報告書の修正、差戻し（講師へ） |
| 営業（sales） | 事務確認後の**最終承認**を行います。 | 最終承認（＝完了）、差戻し（事務へ）、完了後の差戻し |
| 経理（admin_master） | 全報告書の参照・PDF出力、契約管理、ユーザー管理を行います（承認フローの外）。 | 参照、契約管理、ユーザー管理（学校承認スキップ設定を含む）、運用 |
| 管理責任者（admin_chief） | 経理と同等＋管理責任者専用操作。 | 経理と同等＋学校承認スキップの手動実行 など |

承認の流れ: `下書き → 学校承認待ち → 事務確認待ち → 営業確認待ち → 完了`。**営業（sales）が承認した時点で完了**です（経理の最終承認ステップはありません）。学校承認が不要な学校（ユーザー管理で学校承認スキップを設定）では、提出時に学校確認をスキップして事務確認へ進みます。

> 補足: 次のいずれかに該当する報告は、学校確認の前に**事務の事前確認**が入ります（講師 → 事務 → 学校 → 事務 → 営業。事務が承認すると学校承認へ進み、差戻すと講師へ戻ります）。
> 1. 担当業務の月分が契約の月分固定を**超過**している
> 2. 担当業務・副担当業務（分）に**1〜9分単位の手入力**（10分単位でない値。例：151分）がある
>
> 2に該当する報告書は、講師の承認管理画面が4ステップ表示（運営へ依頼→学校へ依頼→学校承認→運営承認）・ボタンが「提出」表記になり、提出時に承認ワークフローが変わる旨のポップアップ（OK／キャンセル）が表示されます。

## 2. ログインとデモアカウント

### ログイン手順

1. ブラウザで `http://localhost:8001/login` を開きます。
2. メールアドレスとパスワードを入力し、`ログイン` を押します。
3. 複数の役割を持つアカウントは、ログイン後に役割選択画面が表示されます。使用する役割を選びます。
4. 役割に応じた初期画面へ移動します（経理・管理責任者は経理キュー `/finance/queue`）。

パスワードを忘れた場合の手順は指導実績報告システムと同じです（リセットメール → リンク → 新パスワード設定）。開発環境のメールは MailHog（`http://localhost:8025`）で確認します。

### デモアカウント

パスワードはすべて `Passw0rd!` です。学校・営業・事務のアカウントは `docker compose exec new_backend python -m app.scripts.create_test_users` で作成します。講師・経理は指導実績報告システムのシードアカウント（`tutor1@example.com` / `master1@example.com`）を共有します。

| 役割 | メールアドレス | 名前 |
|---|---|---|
| 講師 | tutor1@example.com | 講師 一郎（共有） |
| 学校 | school1@example.com | 椅子戸学園 |
| 事務 | office1@example.com | 事務担当一郎 |
| 営業 | sales1@example.com | 営業担当一郎 |
| 経理（管理者） | master1@example.com | 管理者（共有） |

## 3. 講師の操作手順

### 業務連絡表を作成する手順

1. 講師アカウントでログインし、`http://localhost:8001/tutor/reports` を開きます。
2. 画面上部の月選択で対象月を確認します（作成できるのは当月分のみです）。
3. `派遣先事業所の名称・組織単位` のプルダウンから学校を選びます。
   - このプルダウンには、**経理の契約管理に登録された自分の契約校のみ** が表示されます。表示されない場合は経理へ契約登録を依頼してください。
4. 学校を選ぶと、その学校の契約内容（担当業務・副業務・採点の有無など）に応じてフォームの列が切り替わります。
5. 明細行に、日付・担当時限・各業務の分数・休憩・往復交通費・内容を入力します。
   - **担当時限を選ぶと右隣の担当業務（分）へ「コマ数×50分」、休憩時間（分）へ「（コマ数−1）×10分」が自動入力されます**（例：3コマ → 担当業務150分・休憩20分）。自動入力後の分数は1分単位で手動修正できます（担当時限を選び直すと自動値で上書きされます）。
   - **コマ設定（時間割）のある契約**では、業務開始＝選択した最早コマの開始時刻、担当業務（分）＝選択コマの合計時間、休憩時間（分）＝コマ間の隙間が自動入力されます。副担当業務等（分）は既定で**最終コマの後**に実施する扱いです（休憩はそのまま・終了時間が副担当の分だけ延びる。例：①③④コマ＋副担当50分 → 担当150分・休憩80分・終了は④の直後+50分）。コマとコマの間に実施した場合は、休憩時間の右隣の**「副担当の位置」を「コマ間」**へ切り替えると、休憩時間（分）＝隙間−副担当業務等の分で自動再計算されます（実働6時間超45分・8時間超60分の下限は自動で維持）。
   - **業務開始〜終了時間は自動計算です（手動入力不可）**。開始は8:40固定、終了は担当時限より右の時間（分）列（担当業務・副担当業務・採点の分・休憩時間）の合計を開始に加算した時刻です（例：合計170分 → 08:40〜11:30）。分数を修正すると連動して更新されます。
   - 時間（分）の合計が長すぎて終了時間が同日23:59を超える場合は「計算不可」と表示され、保存できません。
   - 同じ日付の行を複数入力することはできません（警告が表示されます）。
   - 採点欄が有効な契約では、採点の「回」「分」を1セルに併記入力します。
   - 有給休暇・欠勤の日は、種別を選んで記録できます（時間は持ちません）。
   - 自己都合・学校行事の日も種別で記録できます。この場合、**担当時限は選択不可・担当業務（分）は0分固定**になり、副業務・採点・休憩・往復交通費・内容は手動で入力できます（勤務日数には数えず「自己都合：N回・学校行事：N回」として集計。入力した分は各列の合計に含まれます）。
   - 日付ボックス内の右側に曜日が `2026/07/07 (火)` の形式で表示されます。
   - **スマホ（画面幅768px未満）では、明細は横スクロールの表ではなく一覧（日付・開始・終了・交通費・事由）で表示されます**。「＋ 日付を追加」または日付のタップで詳細入力画面が開き、1日分（日付・種別・担当時限・各分数・往復交通費・内容）をまとめて入力して `保存` を押すと一覧へ戻ります（自動計算・種別・同日重複などの入力ルールはPCと同じです）。事由欄には有給・欠勤・自己都合・学校行事が表示されます（勤務の日は空欄）。
6. `保存` を押します。一覧に業務連絡表が追加されれば作成完了です（スマホでは `保存` ボタンが画面下部に常時表示されます）。

複数の学校を担当する場合は、`＋別の学校の報告書を作成` ボタンや横タブで学校を切り替えて、学校ごとに並行して作成できます。

### 確認・編集・削除

1. `http://localhost:8001/tutor/reports` を開きます。
2. 横タブで対象の学校（報告書）を選びます。
3. 下書き・差戻し中の報告書は編集できます。
4. 下書き・差戻し中の報告書は `報告を削除` で削除できます（本人分のみ）。
5. 提出済み・過去月の報告書は参照のみです。

### 提出・再提出する手順

1. `http://localhost:8001/tutor/approval`（承認管理）を開きます。
2. 対象の学校・月のカードを確認します。
3. 内容に問題がなければ `承認依頼` ボタンで提出します。押下時に確認ポップアップが表示され、`提出する` で確定します（キャンセルで提出を中止できます）。状態が `学校承認待ち`（または学校承認スキップが設定された学校では `事務確認待ち`、月分超過・1〜9分単位の手入力がある場合は `事務事前確認待ち`）に進みます。1〜9分単位の手入力がある報告書はボタンが「提出」表記になり、確認ポップアップの代わりに承認ワークフローが変わる旨のポップアップが出ます。承認依頼時にリマインド（送付間隔・回数）も設定できます。
4. `報告書一覧` の入力フォーム下部（保存/更新・取消ボタンの右端）にも `承認依頼` ボタンがあり、当月の下書き・差戻し中の報告書はフォームの内容を保存したうえでそのまま提出できます（機能は承認管理の `承認依頼` と同じ）。こちらも押下時（保存後）に確認ポップアップが表示され、キャンセルした場合は提出されません（保存は残ります）。
5. 差戻しがある場合はカードに差戻し理由が表示されます。`報告書一覧` で内容を修正し、`承認依頼` ボタンまたは承認管理から再提出します（再提出時も同じ確認ポップアップが表示されます）。

### 提出後に修正したいとき（差戻しを要求する）

提出後の報告書は差戻されるまで修正できませんが、承認管理のカードから**現在の承認担当（学校または運営）へ差戻しを要求**できます。

1. `承認管理` を開き、対象カードの `差戻しを要求` ボタンを押します（提出後〜最終承認済みまでの間に表示されます。学校確認待ちの過去月分は対象外）。
2. 差戻しを要求する理由を入力して `差戻しを要求する` を押します。カードに「差戻し要求中」バッジと要求理由が表示されます。
3. 承認担当が**許可**すると報告書が差戻され（差戻し理由に要求理由が転記されます）、通常の差戻しと同様に修正・再提出できます。**却下**された場合は却下理由がカードに表示され、必要であれば再度要求できます。
4. 要求中に承認が進んだ場合、要求は消えずに**次の承認担当へ引き継がれます**（例：学校が承認した場合は運営（事務）が要求に対応します）。
5. 差戻し要求のメール通知はありません。急ぎの場合は報告書のチャット等で承認担当へ直接連絡してください。

### 長期休業などで当月授業が無いとき（授業なし申請）

長期休業などでその月の授業を行わない場合は、報告書一覧のヘッダー（対象月セレクトの下）の **`この月は授業なし（長期休業等）`** トグルをONにします（確認ポップアップあり・全契約対象）。

- 申請中は、学校から運営（事務・営業）への「学校承認がすべて完了しました」通知の集計対象から外れます（あなたの報告書が無くても他の講師全員が承認されれば完了通知が送られます）。
- 報告書の作成・提出は申請中も通常どおり可能です。トグルをOFFにすればいつでも解除できます（集計対象に戻ります）。
- 申請は表示中の対象月に対して行われます。月を切り替えると、その月の申請状態がトグルに表示されます。

## 4. 学校の操作手順

1. 学校アカウントでログインし、`http://localhost:8001/school/approval` を開きます。
   - 講師がまだ承認依頼していない下書きは表示されず、内容も参照できません。
2. 講師×月ごとのカード（「YYYY年M月分　学校名：〇〇　講師名：●●」）が表示されます。
3. `報告書を確認` を押すと、別ウィンドウで業務連絡表を参照できます。
4. 内容に問題がなければ `承認する` を押します。状態が事務確認待ちへ進みます。
5. 修正が必要な場合は `差戻し` を押し、理由を入力します。報告書は講師へ差し戻されます。
6. 講師から**差戻し要求**が届いている場合は、カードに「差戻し要求あり」バッジと案内が表示されます。`報告書を確認` で参照画面を開くと要求理由が表示されるので、`要求を許可（講師へ差戻す）` または `要求を却下`（却下理由必須・講師に表示されます）で対応します。要求に気づかず承認した場合、要求は次の承認担当（運営）へ引き継がれます。

## 5. 事務・営業・経理の操作手順

事務（`/office/queue`）・営業（`/sales/queue`）・経理（`/finance/queue`）の3画面は同じ構造で、`タスク`（自分の対応待ち）と `パイプライン`（全体の進捗列）で構成されます。

### 共通操作

1. 各ロールのアカウントでログインし、対応する画面を開きます。
2. `タスク` で自分の対応待ち（自ロールの確認待ち）カードを確認します。
3. `報告書を確認` を押して内容を参照します。
4. 承認または差戻しを行います。

### 各ロールの承認・差戻し

| ロール | 画面 | 承認の効果 | 差戻し先 |
|---|---|---|---|
| 事務 | `/office/queue` | 営業確認待ちへ前進 | 講師（差戻し時） |
| 営業 | `/sales/queue` | **完了（最終承認）** | 事務 |
| 経理 | `/finance/queue` | （承認フローの外。参照・運用・管理） | — |

- **営業の承認で完了（最終承認済み）になります**。経理（admin_master）はこの承認フローには入らず、全報告書の参照・PDF出力・契約管理・ユーザー管理などを担当します。
- 営業からの差戻しは **事務** に届きます（`事務へ差戻し中`）。事務は「承認＝営業へ前進」または「差戻し＝講師へ」を選びます。
- 営業は完了（最終承認済み）後でも、修正が必要な場合は差戻しできます（事務へ戻ります）。
- 事務は、事務確認待ち・営業確認待ち・事務差戻し中の報告書を**修正**できます（再承認は不要。修正内容は講師・学校へ通知されます）。修正モーダルの入力ルールは講師の報告書一覧と同じです（種別による入力制限・担当時限からの担当業務/休憩の自動入力・業務開始〜終了時間の自動計算・同日重複ガード・労基休憩の下限。モーダル下部に勤務日数・種別回数・各列合計の集計が表示されます）。
- パイプライン列の `強制クローズ` で、滞留した報告を終了させることができます（理由必須・当月分は不可。事務／営業／経理／管理責任者が実行可能）。
- 経理（`/finance/queue`）は完了済みを含む全体の進捗を参照できます。
- 講師から**差戻し要求**が届くと、自分がボールを持つ案件は `タスク` に「差戻し要求」バッジ付きで表示されます（あなたの承認待ち件数にも含まれます。事務＝事前確認待ち・事務確認待ち・事務差戻し中、営業＝営業確認待ち・**完了後**）。`報告書を確認` で参照画面を開き、要求理由を確認して `要求を許可（講師へ差戻す）` または `要求を却下`（却下理由必須）で対応します。許可すると報告書は講師へ直接差戻されます。要求対応前に承認を進めた場合、要求は次の承認担当へ引き継がれます（メール通知はありません）。

> 兼務（事務かつ営業）のスタッフは、同じ講師に対して事務の承認・差戻しと営業の承認・差戻しの両方を行うことはできません（どちらか一方のみ。職務分掌）。経理・管理責任者は対象外です。

### 学校の締め日・提出確認メール設定（ユーザ管理・202607161140）

事務・営業・経理・管理責任者は、`ユーザー管理`（`/admin/users`）で学校ユーザーの `詳細` を開くと、**「締め日・提出確認メール設定」**セクションで以下を設定できます。

1. **早期チェック**（ON/OFF）: ONの学校のみ、締め日前の確認メールが自動送信されます。
2. **送信タイミング**: 締め日の何日前に送るか（0〜60日・既定3日前）。
3. **締め日（年間設定）**: `◀ 年 ▶` で年を切り替え、各月分の締め日を日付で設定します（**その月の日付のみ選択可**＝1月分は1月のカレンダーのみ・空欄の月は送信対象外）。`設定を保存` で確定します。

- 早期チェックがONの学校は、締め日のN日前〜締め日当日に1回だけ、**営業担当全員**へタイトル**【至急確認】**の「締め日は〇〇です、提出状況を確認してください」メールが送られます（本文に現在の学校承認状況の内訳つき）。
- 送信済みの月には「送信済」バッジが表示されます。**締め日を変更すると再送対象に戻ります**。全講師の学校承認が完了している学校には送信されません（完了通知で把握済みのため）。
- **CSVで一括設定**もできます（202607161332）: 一覧上部の「学校の締め日設定CSV」で対象年を選んで `CSVエクスポート` → Excel等で編集 → `CSVインポート`。行は学校No×対象年、各月列は締め日の「日」（例: 25）で入力します（**空欄はその月の締め日を削除**・早期チェック/通知日数は空欄なら現状維持・1件でもエラーがあると全件取り込まれません）。詳しい書式はCSVヘルプ（「?」ボタン）を参照してください。

### ユーザー管理一覧の見方・ロール変更（202607171705）

- 一覧の各行は、ロールを**バッジ**で表示します（状態も「有効／無効」のバッジ表示）。
- **ロールの変更（営業・事務の兼務設定）は、行の `詳細` を開いて「ロール設定」のチェックを変えて `保存` します**（以前の一覧上のチェックボックスと「更新」ボタンは廃止し、詳細画面に集約しました）。
- 行の操作ボタンは `詳細`（ユーザー詳細・各種設定・無効化・削除）と `コピー`（下記）の2つです。

### ユーザーをコピーして新規登録（202607171557）

似た設定のユーザーを続けて作るときは、`ユーザー管理` 一覧の各行の **`コピー`** ボタンが便利です。

1. コピー元にしたいユーザーの行で `コピー` を押します。
2. 表示されるウィンドウで、**氏名**と**メール**（どちらも新しいもの）を入力して `コピーして登録` を押します。
3. コピー元の**ロール・利用システム・学校の承認スキップ設定**が引き継がれます。No は自動で割り当てられ、初期パスワードは `Passw0rd!`（初回ログイン時に変更）です。**招待メールは送られません**（そのまますぐ使えるアカウントが作られます）。

- 氏名・メールは、**すでに登録されているものと重複するとエラー**になります（別の氏名・メールを入力してください）。
- 管理責任者アカウントのコピーは、管理責任者のみ可能です。

## 6. 契約管理（経理）

経理（管理者）は `http://localhost:8001/admin/contracts` で、講師×学校ごとの契約を管理します（この画面は営業・事務・管理責任者も開けます）。契約はその講師の業務連絡表フォーム（担当業務・副業務・採点の列）の元になります。

### 契約を1件ずつ登録する手順

1. 経理アカウントでログインし、`/admin/contracts` を開きます。
2. `新規登録` で登録ドロワーを開きます。
3. 講師・学校・お客様ID・弊社担当・契約期間などを入力します。
4. 担当業務（前期）・担当業務（後期）のうち**少なくとも1期**に、委託業務名（設定する期は必須）・委託業務ID・個別契約IDと、期別の月時間（分）・週コマ（任意）・適用期間（設定する期は必須。両期を設定する場合は重複不可）・コマ設定（担当時限の時間割・任意）を入力します（前期だけの契約も登録できます。未設定の期は空欄のままにします）。必要に応じて副業務①〜⑤も入力します。
   - 報告書の担当業務列（1列）・コマ設定は、講師の入力タイミング（今日）がどちらの適用期間内かで自動的に使い分けられます。月時間（分）・週コマを超えて提出された報告書は、学校確認の前に事務の事前確認を挟む承認ルートへ切り替わります。
   - **コマ設定を使用する**チェック（既定ON・担当業務ブロックの上）で、コマ設定の使用/未使用を切り替えられます。**未使用**にすると前期・後期のコマ設定はグレイアウトして編集できなくなり（設定値は保持・再びONで編集可）、この契約の講師の報告書は担当時限なしの手入力方式（業務開始時間・担当業務・副担当業務・休憩時間を手入力→業務終了時間のみ自動計算）になります。
5. 採点を報告書に出す場合は `採点を追加する` をチェックし、採点の項目名・単位・委託業務ID・個別契約IDを入力します。
6. 保存すると、対応する紐付け（assignment）が自動で作成・更新され、**契約管理番号が作成順に自動発番**されます（例 00001。一覧の先頭列と編集ドロワーの基本情報に表示。番号は自動発番のため編集できません）。

### 既存の契約をコピーして登録する（202607171557・202607171705）

同じような契約内容を別の講師×学校で作るときは、契約一覧の各行の **`コピー`** ボタンを使います。

1. コピー元の契約の行で `コピー` を押すと、小さな登録ウィンドウが開きます（コピー元の講師×学校・契約Noが表示されます）。
2. **新しい講師と学校を選んで** `コピーして登録` を押します（同じ講師×学校の契約が既にある場合はエラーになります）。
3. 契約内容（就業場所・担当業務・コマ設定・副業務・任意項目列・表示項目・契約期間など）はコピー元から**すべて引き継がれ**、契約管理番号は新しい契約として**作成順に自動発番**されます。内容を調整したい場合は、登録後に `編集` から変更してください。

### CSV で一括登録する手順

1. `/admin/contracts` 上部の `CSVテンプレート` を押してテンプレート（記入例付き）をダウンロードします。
2. テンプレートに沿って契約を記入します。
   - 講師は **講師番号**、学校は **学校名** で指定します。
   - 記入例・コメント行（講師番号が空、または先頭が `#`）と空行は取り込まれません。
3. `CSVインポート` でファイルを選択してアップロードします（文字コードは UTF-8 / Shift-JIS を自動判定）。
4. 結果モーダルで取り込み件数を確認します。
   - 1件でもエラーがある場合は、行番号付きのエラー一覧が表示され、**全件が中止**されます。エラーを直して再アップロードしてください。
   - 同じ（講師×学校）の契約が既にある場合は上書き（更新）されます。

## 7. 報告書の参照・PDF / CSV 出力

- 各ロールの `報告書を確認` から、別ウィンドウで読み取り専用ビュー（`/reports/{id}/view`）を開けます。契約に応じた動的列（担当業務・副業務・採点）と種別（勤務／有給／欠勤／自己都合／学校行事）がそのまま表示されます。
- 報告書は PDF（A4 横向き）で出力できます。**PDF の列・値・集計は参照ビューと一致**します（契約由来の動的列も反映されます）。
- 経理は対象月・契約・講師の単位で、最終承認済み（完了）の報告書を PDF / CSV でまとめて出力できます。

## 8. 通知・リマインド

- 承認が次の担当へ進むたびに、対象ロールへ承認依頼メールが送られます。
- 学校承認・**最終承認（営業の承認による完了）**・差戻しの際にも、関係者へメールが送られます。
- 事務が報告書を修正した場合は講師・学校へ通知されます（学校承認スキップ校の学校にも送られます。メール本文は修正した担当者の個人名を出さず「イスト事務担当者により修正されました」表記）。差戻し中の報告書を講師が修正した場合は差戻した担当者へ通知されます。
- 講師の**差戻し要求**（要求・許可・却下）はメール通知の対象外です。承認担当への到達経路は各画面のタスク・バッジ表示です（要求を許可して差戻された報告書の再提出時は、通常どおり承認依頼メールが送られます）。
- 1つの学校の**契約講師全員の学校承認が揃った時点**で、事務・営業の全員へ完了メールが送られます（講師の「当月授業なし」申請中の講師は集計対象外。申請で全員承認が成立した場合もその場で送られます・202607161140）。
- **早期チェックがONの学校**は、設定した締め日のN日前〜締め日当日に1回、営業担当全員へ【至急確認】の提出状況確認メールが送られます（ユーザ管理の学校詳細で設定・202607161140）。
- 月末リマインダー（毎日09:00）、未処理報告アラート（毎日06:00）、学校承認リマインド（契約のリマインド設定に従う）が自動で動作します。
- 実メールは送信キューに溜めて1通ずつ間隔をあけて送るため、一括操作・月末でも一度に大量送信されません。開発環境のメールは MailHog（`http://localhost:8025`）で確認できます。

## 9. 起動・停止（管理者向け）

イスト勤怠レポート for EMPS は指導実績報告システムと同じ Docker Compose で同時に起動します。

```bash
# 起動（指導実績報告 backend:8000 / 本システム new_backend:8001 / db / mailhog がまとめて起動）
docker compose up -d --build

# 本システムのみログ確認
docker compose logs new_backend -f

# 本システムのマイグレーション適用（通常は起動時に自動実行）
docker compose exec new_backend alembic upgrade head

# 学校・営業・事務のデモユーザー作成
docker compose exec new_backend python -m app.scripts.create_test_users
```

本システムのマイグレーションは指導実績報告システムと分離して管理されます（バージョンテーブル `work_alembic_version`）。本システムのコンテナは起動時に `alembic upgrade head` を自動実行します。
