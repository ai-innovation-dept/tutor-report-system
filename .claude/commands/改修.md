## 着手手順（要点。詳細は下部 @CLAUDE.md の原文が正）
1. **管理番号を採番**する（依頼者は指定しない）。着手時点の JST を `YYYYMMDDHHMM` で取得し、
   **結果の冒頭に「管理番号: NNNN」**と明記する。⚠️ この環境の Git Bash `date` は UTC を返すため使わない。次のいずれかで取得:
   - PowerShell: `[System.TimeZoneInfo]::ConvertTimeBySystemTimeZoneId([DateTime]::UtcNow,"Tokyo Standard Time").ToString("yyyyMMddHHmm")`
   - or `docker compose exec -T backend python -c "from app.core.time import get_current_jst; print(get_current_jst().strftime('%Y%m%d%H%M'))"`
   - 既存番号の作業の継続なら新規採番せず**元番号を引き継ぐ**（docs/HANDOFF.md・memory・git ログで確認）。
2. 複数項目は**優先順位**をつける。品質が落ちそうなら分割し、実施項目番号・積み残しを明記。
3. **影響範囲・連携**を先に洗い出す（複製ペアの同時更新・タッチポイント全列挙・共有リソース・migration・連携先の改修要否）。
4. デザイン改修は**統一性・見やすさ・操作性・迷わない導線**（既存トークン再利用・2システムパリティ・
   入力フォーム／ポップアップの意匠）を最優先。
5. 要件レベルの不明点は着手前に**文章で確認**。設計の細部は妥当な既定を宣言して進める。
6. **本番メールを送らず**両システムのフル pytest を通す（`bash .claude/hooks/verify.sh` ＝ console・実送信ゼロ・
   成功でpush前マーカー更新）。
7. `docker compose up -d --build`（コード/テンプレ/依存の変更時）→ 関連ファイルのみコミット → `git push`。
8. **完了時セルフチェック**を結果に出力し、「本番未反映」を明記、docs/HANDOFF.md・memory を更新する。
